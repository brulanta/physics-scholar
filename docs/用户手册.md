## 1. 系统要求

| 项目     | 要求                                      |
| -------- | ----------------------------------------- |
| 操作系统 | Windows 10 或 Windows 11（64位）          |
| 网络     | 需要联网（调用 LLM API 和论文检索服务）   |
| 磁盘空间 | 程序本体约 350 MB，另需预留存放论文的空间 |
| 浏览器   | Chrome / Edge / Firefox 均可              |

> macOS 和 Linux 暂不提供打包版本，如需使用请联系开发者。

## 2. 安装与首次启动

### 第一步：下载解压

从发布页面下载 `PhysicsScholar_v1.0.zip`，解压到任意位置（建议放在不含中文的路径下，如 `C:\Tools\PhysicsScholar\`）。

解压后文件夹结构如下：

```
PhysicsScholar/
├── PhysicsScholar.exe    ← 双击这个启动
└── _internal/            ← 运行时文件，不要动
```

### 第二步：启动程序

双击 `PhysicsScholar.exe`。

- 首次启动可能需要 **10–30 秒**初始化，请耐心等待
- 启动成功后，任务栏右下角（系统托盘）会出现程序图标
- 默认浏览器会自动打开 `http://localhost:57321`，显示欢迎界面

> 如果浏览器没有自动打开，请手动在浏览器地址栏输入 `http://localhost:57321`

## 3. 首次配置：填写 API Key

**PhysicsScholar 不内置 API Key**，需要你自己注册并填入。只需配置一次，之后每次启动自动生效。

**进入系统配置的方式：** 点击界面**左下角**的「设置 & 论文库」，在右侧弹出的抽屉栏**右上角**找到齿轮图标（⚙️），点击即可进入系统配置界面。

---

### 3.1 必填：主模型（LLM）

主模型负责理解问题并生成回答。推荐使用 **DeepSeek**（国内访问稳定）。

**注册并获取 Key：**

1. 打开浏览器，访问 [https://platform.deepseek.com](https://platform.deepseek.com)
2. 注册账号并登录，**充值适量额度**（Key 无余额时无法正常使用）
3. 进入「API Keys」页面，点击「新建 API Key」
4. 复制生成的 Key（以 `sk-` 开头）

**在系统配置中填写：**

| 字段        | 填写内容                                           |
| ----------- | -------------------------------------------------- |
| LLM API Key | 粘贴刚才复制的 Key                                 |
| Base URL    | `https://api.deepseek.com`                         |
| 模型        | 点击「获取模型」按钮，从下拉框中选择你想使用的模型 |

> 如果你已有 OpenAI、Gemini 或其他兼容服务的 Key，同样可以使用，按对应服务商的 Base URL 填写即可。请确保所选服务的账户有足够余额。

---

### 3.2 必填：Embedding 模型（硅基流动）

Embedding 模型将论文内容转为可检索的向量，是 RAG 问答的基础。使用**硅基流动**平台（**注册免费，无需充值**）。

**注册并获取 Key：**

1. 打开 [https://siliconflow.cn](https://siliconflow.cn)
2. 注册账号并登录
3. 进入「API 密钥」页面，创建新密钥并复制

**在系统配置中填写：**

| 字段         | 填写内容           |
| ------------ | ------------------ |
| 硅基流动 Key | 粘贴刚才复制的密钥 |

> Embedding 模型已固定为 BAAI/bge-m3，无需选择。

---

### 3.3 选填：副模型（Secondary LLM）

副模型目前主用于 Jina 全文阅读的「分片打分」模式（见第 6.3 节），**不使用该功能则无需填写**。建议选择**价格低、上下文长**的模型，因为该功能会消耗约等于目标论文全文长度的 Token。

---

### 3.4 选填：Jina Key

Jina 用于读取在线论文全文。**不填也可以正常工作**（每分钟最多读取 20 次）。

新用户在 [https://jina.ai](https://jina.ai) 注册后会获赠初始额度，如需长期使用可考虑注册获取 Key 后填入配置。

---

### 3.5 选填：Semantic Scholar (S2) Key

S2 是主力论文检索工具。不填时走公共请求池，填入 Key 后可提升调用频率。

**申请方式：** 前往 S2 官网填写申请表。如果你是在校研究人员，使用 `.edu` 等学校邮箱并在申请中说明科研身份和用途，审核通常很快。Key 无额度限制，但长期不使用可能被注销，具体请参见 S2 官网说明。

---

### 3.6 推荐：OpenAlex Key

OpenAlex 用于文献检索与摘要补全。建议配置 API Key，以提升稳定性和调用额度。

**获取方式：**注册 OpenAlex 账号后，在设置页面复制 API Key 并填入系统配置。

---

### 3.7 保存配置

填写完成后，点击**保存**按钮，**重启程序**后配置生效。

## 4. 界面总览

PhysicsScholar 的界面布局与常见聊天软件类似：

```
┌──────────────┬──────────────────────────────────────┐
│⚛ PhysicsScholar│                                    │
│              │                                      │
│  [会话列表]   │           对话区域                    │
│              │                                      │
│  会话一       │  用户消息气泡（右侧）                  │
│  会话二       │  Agent 回答气泡（左侧）                │
│  会话三       │  ↑ 鼠标悬停时显示操作按钮              │
│  ...         │                                      │
│              │                                      │
│              ├──────────────────────────────────────┤
│[设置&论文库]  │ [🗨讨论]    [输入框]          [发送↑] │
└──────────────┴──────────────────────────────────────┘
```

**各区域说明：**

**左侧栏**

- 顶部显示应用名称「⚛ PhysicsScholar」
- 中部为会话列表，点击切换对话
- 底部为「设置 & 论文库」入口，点击后右侧弹出抽屉

**对话区域**

- 右侧为用户消息，左侧为 Agent 回答
- 鼠标悬停到消息气泡上时，下方会出现操作按钮：
  - **用户消息**：版本切换（有分支时显示 `‹ 1/2 ›`）、发送时间、编辑、复制
  - **Agent 消息**：复制、点赞、点踩、重新生成、版本切换

**底部输入区**

- 左侧：「🗨 讨论」按钮，点击后点亮变为「🗨 讨论中」，切换学术讨论模式
- 中间：消息输入框，按 Enter 或点击右下角发送按钮发送
- 右下角：发送按钮

**设置 & 论文库抽屉**（点击左下角入口弹出）

从上至下依次包含：

- **顶栏**：右侧有⚙️（系统配置）和 ✕（关闭）
- **外观**：主题切换（日间 / 夜间）、字体选择
- **功能**：
  - 严格模式：开启后，论文入库时必须解析出完整的标题、作者、年份三项才返回确认；关闭后只需解析出标题即可
  - 双语对照：开启后，Agent 回答中的 Reference 区域会将引用原文翻译为中文；关闭则保留原文
- **论文库**：
  - 顶栏有上传、搜索、排序（A-Z / Z-A / 年份正序 / 逆序）按钮
  - 下方为已入库论文列表，可滚动浏览，在此处进行删除操作、复制论文doc_id

## 5. 基本使用流程：上传论文并提问

### 第一步：新建会话

点击左侧会话列表上方的「**+**」，创建新会话。

### 第二步：上传论文

点击左下角「设置 & 论文库」，在抽屉的论文库区域点击**上传**按钮（也可直接将 PDF 文件拖拽到页面上）。

- 支持同时选择多个 PDF 文件
- 上传后会弹出确认窗口，显示解析出的论文标题、作者、年份，确认无误后点击「入库」
- 入库完成后，论文出现在抽屉的论文列表中，同时显示入库状态与doc_id

> 论文只需**上传一次**，重启程序后数据依然保留。

### 第三步：提问

在底部输入框中输入问题，按 Enter 或点击发送。

**示例问题：**

- "这篇论文的主要贡献是什么？"
- "帮我解释一下 EOML 的工作原理"
- "帮我搜索最近关于微波光子滤波器的论文，把找到的链接也发出来"

Agent 回答会附带引用标注（行内角标 + 页脚 Reference 列表），引用段落同时展示英文原文和中文译文（如已开启双语对照）。

## 6. 功能详解

### 6.1 RAG 问答（基于你上传的论文）

上传论文入库后，Agent 会从中检索与你问题相关的段落作为回答依据。回答末尾的 Reference 区域列出所有引用来源，可用于核实。

**如果想让 Agent 专注讨论某一篇论文：**

你可以在对话中告知 Agent 你想讨论的论文的名称/作者/年份，Agent 会先在本地库中匹配对应的论文 ID，再定向检索该论文内容。

如果需要更精确地锁定某篇论文，可以在「设置 & 论文库」的论文列表中找到该论文，复制其 **doc_id**，然后直接告知 Agent：「请用 doc_id 为 `xxxxxx` 的论文回答我的问题」，Agent 会精确定向到这篇论文。

---

### 6.2 在线论文检索

在对话中直接说「帮我搜索……」，Agent 会自动调用 Semantic Scholar、OpenAlex、arXiv 等工具进行检索。

**实用技巧：** 如果你之后想用右键快捷入库功能，或者需要论文链接，记得在提问时说明：

> "帮我搜索关于 optical frequency comb 的最新综述，**把找到的论文链接也发出来**"

Agent 默认专注于回答问题，不一定主动输出链接，明确要求后会一并附上。

---

### 6.3 Jina 全文阅读

当检索到的摘要不足以回答问题时，Agent 可以通过 Jina 读取论文全文。**前提是对话上下文中存在可访问的 PDF 链接**（来自 Agent 的检索结果，或者你自己提供）。

Jina 有两种运行模式，用你的提问方式来控制：

**模式一：读取开头片段**（不消耗副 LLM）

> "帮我读一下这篇论文的开头，了解一下大概"
> "先读一下这篇文章的摘要和引言部分"

适合快速了解论文结构和研究方向。

**模式二：全文分片打分（定向召回）**（消耗副 LLM，等待时间较长）

> "这篇论文的实验部分用了什么数据集？"
> "这篇文章关于 XX 的方法细节是什么？"

带着具体问题提问时，Agent 会读取全文并定向召回相关片段。**注意：此模式消耗的 Token 量约等于目标论文全文长度**，建议在副 LLM 额度充足时使用。

**💡 建议：** 如果你需要多次查询同一篇论文的内容，将 PDF 入库后通过 RAG 检索是更经济的方式——入库一次，之后无限次快速召回，不消耗额外额度。

---

### 6.4 右键快捷入库

如果 Agent 的回答中包含 PDF 链接，将鼠标移到该消息气泡上并**右键点击**，前端会尝试识别回答中的链接并弹出操作菜单：

- **每个被识别的链接**旁有两个按钮：
  - 「下载」：在新窗口打开该链接（可用于确认链接是否直达 PDF 文件）
  - 「入库」：将该链接的论文直接入库（按钮灰色表示不可入库或已入库）
- **菜单顶部**有「一键入库」，将所有可入库的链接批量入库

> ⚠️ **注意：右键入库不会弹出元数据确认窗口，论文直接入库。**
>
> 由于部分 PDF 链接可能跳转到出版商页面或人机验证，建议**先点击「下载」确认链接能直接打开 PDF 文件**，再点击「入库」，以避免垃圾内容被入库。

**关于识别率：** 前端对常见 PDF 链接格式做了适配，但网络上的链接格式多样（尤其是 Semantic Scholar 返回的链接），**部分链接可能点开确实是 PDF，但右键菜单未能将其识别**。遇到这种情况属正常现象，可以点击链接在新窗口打开后手动下载，再拖拽上传入库。

---

### 6.5 学术讨论模式

点击输入框左侧的「🗨 讨论」按钮，按钮点亮变为「🗨 讨论中」，进入学术讨论模式。

在此模式下，Agent 会化身批判性的讨论伙伴，主动追问你的论证逻辑、指出潜在问题、引用反例，适合整理研究思路、准备答辩或 Rebuttal。

---

### 6.6 代码辅助

当问题涉及数值计算或仿真时，Agent 会在回答中生成 MATLAB 或 Python 代码，代码有格式规范要求，输出后可直接使用。代码块右上角有**复制按钮**，一键复制。

> 注意：Agent 生成代码后不会在服务端执行，需要你自己在本地运行验证。

## 7. 会话管理

左侧会话列表支持以下操作：

| 操作     | 方法                                       |
| -------- | ------------------------------------------ |
| 新建会话 | 点击列表上方的「+」                        |
| 切换会话 | 点击列表中的会话名                         |
| 重命名   | 点击会话旁的 🖊 笔头图标，输入新名称后回车 |
| 删除会话 | 点击会话旁的删除图标，二次确认后删除       |

> 会话历史保存在本地，重启程序后不会丢失。

**消息分支：** 点击 Agent 回答下方的「🔁重发」，或编辑用户消息后重新发送，新旧版本都会保留。消息气泡上会出现 `‹ 1/2 ›` 切换控件，可在不同版本之间比较。

## 8. 系统托盘与重启

程序启动后在任务栏右下角（系统托盘）显示图标。**右键图标**可以：

- **重启**：重新加载程序
- **退出**：完全关闭程序

### 重启说明

点击托盘「重启」后：

1. 当前浏览器窗口会显示**灰色遮罩**，提示服务不可用，这是正常现象
2. 程序在后台重新启动，约需 10–30 秒
3. 重启完成后**会弹出一个新的浏览器窗口**
4. **可以放心关闭旧窗口，在新窗口中继续使用**

## 9. 常见问题

**Q：双击 exe 没有反应，或者浏览器打不开？**

等待 30 秒，首次启动较慢。如仍无反应，检查任务栏托盘是否已有程序图标——若有，手动在浏览器输入 `http://localhost:57321`。若托盘也没有，检查杀毒软件是否拦截（尝试暂时关闭后重试）。

---

**Q：提问后 Agent 一直转圈不回答？**

通常是 API Key 问题或网络不通。检查步骤：

1. 进入系统配置，确认 LLM Key 和 Base URL 填写正确
2. 确认**账户余额充足**（Key 有效但余额为零同样无法使用）
3. 确认电脑可以正常访问外网

---

**Q：Key 填错了怎么办？**

进入系统配置，直接修改后保存，重启程序生效。

---

**Q：上传论文失败？**

可能原因：

- PDF 有加密保护（部分期刊 PDF 限制解析），可尝试用其他工具去除密码后重新上传
- PDF 文件损坏
- 硅基流动 Key 填写有误（入库需要调用 Embedding API）

---

**Q：重启程序后，之前上传的论文还在吗？**

在。论文数据保存在 `_internal/data/` 文件夹中，程序重启或电脑重启均不会丢失。

---

**Q：切换会话后无法发消息？**

当 Agent 正在处理某个会话的请求时，切换到其他会话后暂时无法发送消息或进行编辑、重发等交互操作，等当前请求完成后恢复正常。

---

**Q：论文已入库，但 Agent 好像没有用到它回答？**

可能是问题与该论文相关性较低，Agent 没有检索到相关段落。可以尝试：

1. 在问题中明确提及论文名称或相关元信息，Agent 会尝试在本地库中匹配并定向检索
2. 在「设置 & 论文库」的论文列表中找到该论文，复制其 **doc_id**，直接告诉 Agent：「请用 doc_id 为 `xxxxxx` 的这篇论文回答我的问题」，Agent 会精确锁定该论文作答

---

**Q：程序占用端口 57321 冲突怎么办？**

关闭占用该端口的其他程序后重试。可在命令行输入以下命令查找占用进程：

```
netstat -ano | findstr :57321
```

## 10. 注意事项

**关于 API 费用**

- 主 LLM 按 Token 计费，日常学术问答消耗不大，请确保账户有余额
- Jina 全文分片打分模式消耗较多 Token（约等于目标论文全文长度），按需使用
- 硅基流动 Embedding 免费，无需充值
- OpenAlex 需注册并配置免费 Key，其提供的每日基础额度足够覆盖日常科研检索，无需额外付费
- arXiv、Semantic Scholar、OpenAlex 检索完全免费

**关于数据安全**

- 上传的 PDF 和对话记录仅保存在本机 `_internal/data/` 文件夹中，不会上传至任何服务器
- 发送给 LLM API 的内容（问题 + 检索到的论文段落）会经过服务商服务器处理，如有保密需求请注意

**关于 Embedding 模型**

- 程序固定使用 BAAI/bge-m3，请勿修改配置文件中的 Embedding 相关设置
- 若未来版本升级了 Embedding 模型，已入库的论文需要重新上传入库

**关于种子论文库**

- 当前版本不包含预置论文，需要用户自行上传
- 即使没有上传论文，在线检索功能（arXiv / S2 / OpenAlex）仍然完全可用

**备份建议**

- 如果积累了大量论文和对话记录，建议定期备份 `_internal/data/` 文件夹
